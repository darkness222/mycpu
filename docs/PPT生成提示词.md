# PPT生成提示词

---

## 提示词正文

请帮我制作一个**开题报告PPT**，共8页，适用于5分钟答辩演讲。

---

### 整体风格要求

- **配色方案**：浅色系主调，整体清新明亮
  - 主色：浅蓝灰色（如 `#ECF0F1` 或 `#F8F9FA`）作为背景
  - 强调色：科技蓝（`#3498DB` 或 `#2980B9`）用于标题和重点
  - 辅助色：浅绿（`#2ECC71`）用于"已完成"状态，橙色（`#E67E22`）用于"进行中"状态
  - 警示色：红色（`#E74C3C`）仅用于强调"核心亮点"等关键信息
  - 避免：纯黑背景、刺眼的高饱和度颜色、过深的暗色调
- **字体**：无衬线字体（如微软雅黑、思源黑体），标题加粗，正文 Regular
- **布局**：留白充足，每页信息量适中，不要堆砌文字
- **风格**：科技感、专业、简洁，适合学术答辩

---

### 第1页：标题页

**内容**：
- 主标题：从0实现一个可运行程序的RISC-V指令集模拟器
- 副标题：计算机系统结构课程项目实践
- 成员信息：XX
- 可添加简单的CPU芯片或电路相关的装饰图标

**设计要求**：
- 背景用浅灰白渐变或纯浅色
- 主标题居中，字号大（36-44pt），蓝色加粗
- 副标题在主标题下方，字号稍小（20-24pt），灰色
- 底部放置成员信息

---

### 第2页：项目亮点总览

**内容**：
- 标题：项目亮点
- 两个核心亮点并列展示：

**亮点一：精美直观的前端**
- 图标：🏗️ 或 前端框架图标
- 关键词：React + Vite | 彩色图表 | 实时渲染

**亮点二：流水线 vs 非流水线一键对比**
- 图标：⚡ 或 流水线图标
- 关键词：一键切换 | 对比教学 | 核心创新

**设计要求**：
- 左右两栏布局，或上下分区
- 每个亮点用一个卡片/色块承载
- 用图标或小图示增强视觉

---

### 第3页：前端展示（核心亮点）

**内容**：
- 标题：精美直观的前端 — 数据驱动的实时渲染
- 要点（ bullet list）：
  - 前后端分离：React 前端 + C++ 后端
  - 流水线可视化：彩色图表实时展示 5 级流水线
  - 寄存器、内存、外设状态实时监控
  - HTTP RPC 通信，后端每执行一条指令主动推送状态
- 可预留一个模拟截图区域（用色块标注"截图区域"）

**设计要求**：
- 左侧放要点文字，右侧预留一个手机/网页框架图形表示前端界面
- 或者用 2-3 个卡片横向排列展示功能模块

---

### 第4页：流水线对比（核心创新点）

**内容**：
- 标题：流水线 vs 非流水线 — 看得见的教学
- 重点强调：这是核心创新！

**对比表格**：
| 模式 | 执行方式 | 特点 |
|------|----------|------|
| 非流水线 | 指令逐条执行，一条完成再下一条 | 简单直观，适合教学 |
| 流水线 | 5级并行：IF→ID→EX→MEM→WB | 高效率，有冒险 |

**核心理念**：
- 教科书上的流水线是静态图片，学生看完还是懵
- 我们的方案：同一个程序，一键切换，非流水线 vs 流水线并行执行
- 学生亲眼看到流水线如何"并行"工作，效率提升在哪里

**设计要求**：
- 用两个并排的流程图示意：
  - 左：IF → ID → EX → MEM → WB（一条线）
  - 右：IF₁ → ID₁ → EX₁ → MEM₁ → WB₁ 同时 IF₂ → ID₂ → ...（多条线并行）
- 用颜色区分不同阶段（如IF用蓝色，ID用绿色等）

---

### 第5页：开发故事

**内容**：
- 标题：开发故事

**故事一：AI辅助开发**
- 困境：RISC-V 指令集37条，细节繁多
- 我们用AI：生成代码框架、理解规范文档、代码审查
- 教训：AI是助手，核心逻辑必须自己搞定

**故事二：解决"伪后端"问题**
- 问题：初期后端只是"仓库"，数据是死的
- 解决：设计RPC协议，后端主动推送，前端实时渲染
- 效果：前后端真正打通，数据是"活"的

**设计要求**：
- 左右分栏或时间线布局
- 用不同颜色区分两个故事
- 可用对话气泡或箭头表示"问题→解决"的转化

---

### 第6页：技术架构

**内容**：
- 标题：技术方案
- 简短概述，不需要过多细节

**架构图**（可用简单层级图）：
```
┌─────────────────────────────────────┐
│     React 前端（Web可视化）          │
│   流水线图、寄存器、内存、外设状态     │
└─────────────────────────────────────┘
              │ HTTP RPC
              ▼
┌─────────────────────────────────────┐
│     C++ 后端（高性能模拟）           │
│   CPU、内存、外设、汇编器             │
└─────────────────────────────────────┘
```

**技术栈**：
- RISC-V RV32I（37条指令）
- 5级流水线
- MMIO外设（UART、Timer、Interrupt）

**设计要求**：
- 用矩形色块和箭头绘制简单架构图
- 技术栈用小标签/徽章样式展示

---

### 第7页：开发进度

**内容**：
- 标题：开发进度

**进度展示**（甘特图或进度条形式）：

| 阶段 | 状态 |
|------|------|
| 核心框架 | ✅ 已完成 |
| 5级流水线基础 | ✅ 已完成 |
| 冒险检测与转发路径 | ✅ 已完成 |
| 外设接口完善 | ⏳ 进行中 |
| 前端美化 | ⏳ 进行中 |

**设计要求**：
- 用横向进度条或时间线形式展示
- 已完成用绿色勾选，进行中用橙色圆圈
- 简洁清晰，不要过于复杂

**设计要求**：

4. **中文字体**：确保中文显示正常，避免字体缺失问题
