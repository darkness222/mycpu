# 《计算机系统结构》项目开题报告

## myCPU：从 0 实现一个可运行程序的 RISC-V 指令集模拟器

---

## 一、项目背景与意义

### 1.1 课程改革背景

《计算机系统结构》课程传统教学模式以理论讲授为主，学生难以直观理解 CPU 的工作原理。本项目响应课程改革号召，采用"项目制"教学方式，让学生通过实际动手实现一个完整的 CPU 模拟器，深入理解计算机系统结构的核心概念。

### 1.2 项目意义

- **理论联系实际**：将课堂所学的指令集、流水线、冒险处理等知识应用于实际实现
- **系统性训练**：涵盖 CPU 设计、汇编语言、外设接口、软件架构等多个方面
- **可扩展性**：项目架构支持后续扩展到操作系统等课程内容

### 1.3 对标参考

本项目对标成熟的开源模拟器项目 [Ripes](https://github.com/mortbopet/Ripes)，同时在教学可视化方面做了针对性优化。

---

## 二、项目目标

### 2.1 总体目标

实现一个基于 RISC-V 指令集的 5 级流水线 CPU 模拟器，具备以下能力：

- 执行 RISC-V RV32I 基础指令集
- 可视化展示流水线执行过程
- 支持汇编程序输入和执行
- 提供外设仿真接口

### 2.2 分阶段目标

| 阶段 | 时间 | 目标 |
|------|------|------|
| 第一阶段 | 第 3-4 周 | 完成项目框架搭建、C++ 核心模块实现 |
| 第二阶段 | 第 5-6 周 | 实现 5 级流水线、冒险检测 |
| 第三阶段 | 第 7 周 | 完成外设接口、前端可视化 |
| 第四阶段 | 第 8 周 | 集成测试、演示视频录制 |

## 三、技术方案

### 3.1 架构设计

采用前后端分离架构：

```
┌──────────────────────────────────────────────────────┐
│                    React 前端                         │
│   (流水线可视化、寄存器/内存视图、程序编辑器)           │
└──────────────────────────────────────────────────────┘
                          │ HTTP/WebSocket
                          ▼
┌──────────────────────────────────────────────────────┐
│                   C++ 后端模拟器                       │
│   (CPU、内存、外设、汇编器)                            │
└──────────────────────────────────────────────────────┘
```

### 3.2 C++ 后端模块

| 模块 | 职责 |
|------|------|
| CPU | 指令执行、流水线控制、冒险检测 |
| RegisterFile | 32 个通用寄存器管理 |
| Decoder | RISC-V 指令解码 |
| Memory | 16KB 物理内存模拟 |
| Bus | 地址空间管理、设备连接 |
| UART | 串口输出仿真 |
| Timer | 定时器中断仿真 |
| Assembler | 汇编到机器码转换 |
| RpcServer | 与前端通信 |

### 3.3 指令集支持

**RV32I 基础指令（37 条）：**

- 算术运算：add, sub, addi
- 逻辑运算：and, or, xor, andi, ori, xori
- 移位运算：sll, srl, sra, slli, srli, srai
- 比较运算：slt, sltu, slti, sltiu
- 内存访问：lw, sw, lb, lbu
- 分支跳转：beq, bne, blt, bge, jal, jalr
- 伪指令：li, nop, halt
- 控制寄存器：lui, auipc

### 3.4 流水线设计

采用经典的 5 级流水线：

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│  FETCH  │ → │ DECODE  │ → │ EXECUTE │ → │ MEMORY  │ → │WRITEBACK│
└─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘
```

**冒险处理策略：**

- 数据冒险：通过转发路径解决
- Load-Use 冒险：插入 stall 周期
- 控制冒险：分支预测 + flush

## 六、风险评估与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| RISC-V 指令复杂度高 | 实现周期长 | 优先实现基础指令，扩展指令后续添加 |
| 冒险处理逻辑复杂 | 可能出现 bug | 充分测试，分模块验证 |
| 前后端通信问题 | 集成困难 | 预留充足时间调试 |
