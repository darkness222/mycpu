# myCPU 技术设计文档

## 1. 整体架构

myCPU 采用前后端分离架构：

```
┌─────────────────────────────────────────────────────────────┐
│                        React 前端                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────────────┐ │
│  │ 编辑器   │  │数据通路  │  │寄存器表  │  │   外设面板      │ │
│  └─────────┘  └─────────┘  └─────────┘  └─────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                              │ JSON/RPC
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      C++ 后端模拟器                           │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────────────┐ │
│  │   CPU   │  │ Memory  │  │   Bus   │  │     Devices     │ │
│  └─────────┘  └─────────┘  └─────────┘  └─────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 CPU 类设计

```cpp
class CPU {
public:
    void reset();        // 重置处理器
    void step();         // 执行一个流水线阶段
    void run(uint64 cycles);  // 运行指定周期数
    void loadProgram(const std::vector<uint32>& program, uint32 addr);

    // 状态访问
    uint32 getPC() const;
    CpuState getState() const;
    PipelineStage getCurrentStage() const;

private:
    void fetch();        // 取指阶段
    void decode();       // 译码阶段
    void execute();      // 执行阶段
    void memoryAccess(); // 访存阶段
    void writeback();    // 写回阶段

    void detectHazards(); // 冒险检测
    int32 alu(int32 op1, int32 op2, uint8 funct3, bool is_sub);

    uint32 pc_;
    RegisterFile registers_;
    Decoder decoder_;
    PipelineRegisters pipeline_regs_;
    CpuStats stats_;
    HazardSignals hazard_signals_;
};
```

### 2.3 流水线寄存器

每个流水线阶段之间有寄存器锁存：

```cpp
struct PipelineRegisters {
    // IF/ID
    uint32 if_id_pc;
    Instruction if_id_instruction;
    bool if_id_valid;

    // ID/EX
    uint32 id_ex_pc;
    Instruction id_ex_instruction;
    int32 id_ex_alu_result;
    int32 id_ex_reg_data1;
    int32 id_ex_reg_data2;
    bool id_ex_valid;

    // EX/MEM
    uint32 ex_mem_pc;
    Instruction ex_mem_instruction;
    int32 ex_mem_alu_result;
    int32 ex_mem_mem_write_data;
    bool ex_mem_mem_read;
    bool ex_mem_mem_write;
    bool ex_mem_valid;

    // MEM/WB
    uint32 mem_wb_pc;
    Instruction mem_wb_instruction;
    int32 mem_wb_alu_result;
    int32 mem_wb_mem_data;
    bool mem_wb_reg_write;
    bool mem_wb_valid;
```


## 3. 冒险检测与处理

### 3.1 数据冒险 (RAW)

当一条指令需要读取上一条指令的结果时发生：

```
add x1, x2, x3   ; x1 = x2 + x3
add x4, x1, x5   ; 需要 x1 的值 ← RAW 冒险
```

### 3.2 Load-Use 冒险

当 Load 指令的结果被下一条指令使用时：

```
lw x1, 0(x2)     ; x1 = MEM[x2+0]
add x3, x1, x4   ; 需要 x1 ← Load-Use 冒险
```

解决方案：插入一个 stall 周期

### 3.3 控制冒险

分支指令可能改变 PC：

```
beq x1, x2, label ; 如果相等则跳转
```

解决方案：分支预测（默认不跳转）+ flush

## 4. 前端设计

### 4.1 组件结构

```
App
├── Header (标题、控制按钮)
├── StatsBar (PC、周期、指令数等)
├── EditorPanel (汇编编辑器)
├── DataPathPanel (流水线可视化)
│   ├── StageIndicators
│   ├── PipelineRegisters
│   └── HazardSignals
├── ProgramPanel (指令列表)
├── RegisterPanel (寄存器文件)
├── MemoryPanel (分段内存视图)
├── TracePanel (执行轨迹)
├── PeripheralPanel (外设状态)
└── RoadmapPanel (后续计划)
```

### 4.2 状态管理

```javascript
const [sim, setSim] = useState(() => createInitialState(program));
const [program, setProgram] = useState(() => assembleProgram(DEFAULT_PROGRAM));
const [isRunning, setIsRunning] = useState(false);
```

### 4.3 执行模式
